# Логи и логирование

## Уровни логирования

- **FATAL** — Фиксирует критические ошибки, приводящие к немедленному завершению процесса. В Android таким уровнем маркируется `FATAL EXCEPTION`.
- **ERROR** — фиксирует события, которые привели к необрабатываемому исключению и аварийному завершению (crash). Именно на этом уровне будет записано само исключение (тип, сообщение, stack trace).
- **WARN** — может содержать предупреждения, которые возникли незадолго до crash (например, нехватка памяти, медленная работа сети, неожиданные данные). Они помогают понять контекст.
- **DEBUG** — полезен для понимания последовательности действий пользователя и вызовов методов непосредственно перед crash (какие данные запрашивались, какие ответы приходили). В production-сборках DEBUG-логи обычно отключены, но в тестовой сборке они могут присутствовать.
- **INFO** — даёт общее представление о жизненном цикле экрана истории операций (открытие, загрузка данных, отображение). Может помочь понять, на каком именно шаге произошёл сбой.
 
*Crash происходит при конкретном действии (просмотр истории). ERROR-лог даст точку отказа (класс, метод, строка). WARN и DEBUG укажут на предпосылки (например, пустой ответ от сервера, некорректная дата, попытка отобразить большое количество элементов). Без этих уровней разработчик будет гадать, что именно вызвало исключение.*

## Механизм получения логов и базовые шаги

Для получения логов используется **Android Debug Bridge (ADB)** через **Logcat** (встроенный в Android Studio или командную строку).

**Базовые шаги:**

1. Подключить тестовое устройство (или эмулятор) с отладочной версией приложения к компьютеру.
2. Включить режим разработчика и отладку по USB на устройстве.
3. Запустить Android Studio и открыть вкладку **Logcat** (или использовать команду `adb logcat` в терминале).
4. Воспроизвести проблему: открыть приложение онлайн-банка, перейти к истории операций, выбрать период «последний месяц» и дождаться crash.
5. После crash остановить запись логов (или сразу скопировать последние строки).
6. Сохранить логи в файл или выделить фрагмент, содержащий стек ошибки.

## Фильтры в Logcat для быстрого поиска причины

В Logcat (Android Studio) можно применить следующие фильтры:

- **По имени пакета приложения**  
  `package:com.example.bankapp`  
  *Позволит видеть только логи целевого приложения, отсекая системные сообщения.*

- **По уровню логирования**  
  Установить уровень `Error` (или `Warn`) – отображаются только ERROR и выше.  
  *Быстро покажет логи определенного типа.*

- **По ключевым словам**  
  В поле поиска ввести:  
  - `FATAL EXCEPTION` – стандартная метка Android для crash.  
  - `HistoryActivity` (или название вашего экрана истории) – сузит до логов, связанных с этим экраном.  
  - `OutOfMemoryError` или `NullPointerException` – типы возможных исключений.  
  - `onCreate` / `onResume` – жизненный цикл.  

- **По процессу**  
  В фильтре `Process ID` выбрать PID вашего приложения (если известно).  
  *Избавляет от логов других процессов.*

- **По тегам**  
  Если разработчики использовали свои теги (например, `HistoryAdapter`, `NetworkManager`, `DBHelper`), можно отфильтровать по ним.
