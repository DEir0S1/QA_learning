# Pairwise (попарное) тестирование

## 1. Входные параметры анкеты

Анкета состоит из 4 параметров:

1. **Возраст**:
   - 18–35
   - 36–55
   - 56–75

2. **Наличие любых задолженностей**:
   - да
   - нет

3. **Тип трудовых отношений**:
   - ИП
   - Самозанятый
   - Бессрочный договор

4. **Наличие действующих кредитов**:
   - да
   - нет

---

## 2. Количество всех возможных комбинаций

Посчитаем количество значений у каждого параметра:

- Возраст: 3
- Задолженности: 2
- Тип трудовых отношений: 3
- Действующие кредиты: 2

Общее количество всех возможных комбинаций: **3 × 2 × 3 × 2 = 36 комбинаций**

Это полный перебор (exhaustive testing), который часто избыточен для тестирования бизнес-логики.

---

## 3. Комбинации по технике уникальных пар (Pairwise)

Техника уникальных пар (pairwise testing) требует, чтобы **каждая пара значений любых двух параметров встретилась хотя бы один раз** в наборе тестов.

Ниже приведён пример набора комбинаций, покрывающего все уникальные пары.

| № | Возраст | Задолженности | Тип трудовых отношений | Действующие кредиты |
|---|--------|---------------|------------------------|---------------------|
| 1 | 18–35  | да            | ИП                     | да                  |
| 2 | 18–35  | нет           | Самозанятый            | нет                 |
| 3 | 18–35  | да            | Бессрочный договор     | нет                 |
| 4 | 36–55  | да            | Самозанятый            | да                  |
| 5 | 36–55  | нет           | ИП                     | нет                 |
| 6 | 36–55  | нет           | Бессрочный договор     | да                  |
| 7 | 56–75  | да            | Бессрочный договор     | да                  |
| 8 | 56–75  | нет           | Самозанятый            | да                  |
| 9 | 56–75  | да            | ИП                     | нет                 |

Данный набор существенно меньше полного перебора (9 вместо 36), но при этом покрывает все возможные пары параметров.

---

## 4. Проверка полноты и корректности набора уникальных пар

### 4.1 Проверка полноты

Чтобы убедиться, что набор полон:

1. Выбираем любые два параметра (например, «Возраст» и «Тип трудовых отношений»).
2. Выписываем все возможные пары их значений (3 × 3 = 9 пар).
3. Проверяем, что каждая из этих пар встречается хотя бы в одной тестовой комбинации.
4. Повторяем шаги для всех пар параметров:
   - Возраст × Задолженности
   - Возраст × Тип трудовых отношений
   - Возраст × Действующие кредиты
   - Задолженности × Тип трудовых отношений
   - Задолженности × Действующие кредиты
   - Тип трудовых отношений × Действующие кредиты

Если для каждой пары параметров все сочетания значений присутствуют — покрытие pairwise считается полным.

---

### 4.2 Проверка корректности

Корректность набора можно проверить следующими способами:

- **Отсутствие несуществующих значений** — используются только допустимые варианты из анкеты.
- **Логическая валидность** — нет противоречивых состояний (если такие ограничения заданы бизнес-логикой).
- **Ручная или автоматическая валидация**:
  - сравнение с матрицей всех возможных пар;
  - использование инструментов генерации pairwise-наборов для сверки результата.
