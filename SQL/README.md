# SQL

**Day00** - Учился создавать SQL-запросы с использованием внутренних подзапросов в частях FROM и SELECT, а также фильтровать данные по диапазону дат.

**Day01** - Осваивал использование множеств и операторов объединения, создание подзапросов, сортировку по нескольким столбцам, а также разные виды соединений таблиц.

**Day02** - Осваивал понимание реляционной алгебры, использование различных типов соединений (JOIN), создание запросов с помощью CTE, обработку пропущенных значений, фильтрацию и сортировку данных.

**Day03** - Учился извлекать данные из базы с помощью запросов и изменять их при помощи языка управления данными (DML) — вставлять, обновлять и удалять записи для поддержания актуальности базы.

**Day04** - Учился создавать и использовать представления (views) для фильтрации данных, генерировать временные диапазоны и находить пропущенные даты, работать с операциями над множествами, рассчитывать скидки и управлять материализованными представлениями, включая их обновление и удаление.

**Day05** - Решенал классическую задачи коммивояжёра (Traveling Salesman Problem). Освоил навыки моделирования графов и маршрутов в SQL, научился создавать таблицы с узлами и стоимостью перемещений а также писать запросы для поиска оптимальных путей с минимальными затратами и анализа результатов с сортировкой.

**Day06** - Научился создавать разные типы индексов, анализировать запросы и оптимизировать их

**Day07** - Научился реализовывать бизнес-требования, обеспечивать целостность данных через ограничения и индексы, оптимизировать производительность БД, работать с последовательностями и массовыми операциями, а также документировать структуру базы

**Day08** - Освоил агрегацию данных (расчет сумм, средних, минимума/максимума), группировку по атрибутам, фильтрацию результатов, соединение таблиц и работу с условиями.

**Day09** - Изучил работу транзакций и уровней изоляции в PostgreSQL а так же аномалии (Lost Update, Dirty Reads, Non-repeatable Reads, Phantom Read, Read/Write Skew и т.д) и взаимоблокировку (deadlock)

**Day10** - Освоил создание функций (SQL/PLpgSQL) и триггеров для реализации бизнес-логики на уровне СУБД, научился разрабатывать систему аудита изменений данных и оптимизировать запросы.

**model.sql** - Модель БД, с которой работал во время обучения.
